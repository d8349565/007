Additional rules for FINANCIAL_METRIC:
- metric_name (e.g. 营业收入, 净利润, 毛利率) must go into qualifiers.metric_name.
- yoy and qoq percentages must go into qualifiers.yoy or qualifiers.qoq, NOT into value_num.
- value_num should be the absolute metric value, not the growth rate.
- **Distinguish cumulative/total values from incremental/change values:** If the evidence says "累计销售收入106.51亿元，较上一年增加16.63亿元", then value_num must be 106.51 (the total), and 16.63 should go into qualifiers.change_amount. Do NOT use the change/increment as value_num.
- Use distinct predicates: "累计销售收入" or "销售收入为" for the total value; "同比增长" / "同比下降" for growth rates.
- If the metric is a ratio (e.g. 毛利率 35%), unit should be "%".
- If the metric is a monetary amount, currency must be specified (CNY/USD/EUR). Default to CNY when the unit contains "元" without other currency indicator (e.g. "亿元" → CNY).
- report_scope (e.g. 合并口径, 母公司口径) goes into qualifiers.report_scope if stated.
- is_forecast should be true if the value is a forecast/estimate/guidance.
- Do not confuse revenue (营收) with profit (利润) or cost (成本).
- If the value is approximate (contains "约", "约合", "大约", "近"), set qualifiers.is_approximate to true.

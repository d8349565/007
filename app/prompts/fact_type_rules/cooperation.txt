Additional rules for COOPERATION:
- **subject**: The initiating/primary party in the cooperation. MUST be a company/organization name.
- **object**: The partner/counterpart company. MUST be a company/organization name. For multi-party cooperation (A + B + C), use the most prominent entity as subject, list partners in object (comma-separated if needed), or create separate records per partnership.
- Both subject and object must be clearly identified from the evidence. If either party cannot be identified, do NOT extract this record.
- cooperation_type goes into qualifiers: strategic_cooperation, joint_development, supply_agreement, framework_agreement, equity_cooperation.
- Distinguish between: "达成合作意向" (intention) vs "已签约" (signed) vs "正式启动" (started) vs "完成" (completed). Put stage in qualifiers.
- scope goes into qualifiers if stated (e.g. 强化港澳及大湾区渠道网络).
- duration goes into qualifiers if stated.
- Do not exaggerate the cooperation strength beyond what the evidence states.
- Supply relationships (供应关系) and general cooperation (合作) should use different predicates.

## CORRECT example:
Evidence: "完成与香港万昌五金建材有限公司、康宝香港有限公司的股权合作，强化港澳及大湾区渠道网络"
```json
{
  "fact_type": "COOPERATION",
  "subject": "东方雨虹",
  "predicate": "完成股权合作",
  "object": "香港万昌五金建材有限公司、康宝香港有限公司",
  "qualifiers": {"cooperation_type": "equity_cooperation", "scope": "强化港澳及大湾区渠道网络"}
}
```

## Predicate 完整性约束
- predicate must be a COMPLETE action phrase, NEVER just "与" or "和".
  ✅ "共同投资设立" / "合作使用" / "代理销售" / "完成股权合作"
  ❌ "与" / "和" / "合作" (too vague)
- If the evidence says "A与B合作成立C", predicate should be "合作成立", NOT "与".
- cooperation_type in qualifiers must EXACTLY match the cooperation nature described in the evidence:
  - "合作使用和应用" → "technology_cooperation", NOT "supply_agreement"
  - "共同投资设立" → "equity_cooperation"
  - "代理销售/授权销售" → "distribution_agreement"
  - "供应涂料/提供产品" → "supply_agreement"
  - Do NOT use "supply_agreement" when the evidence describes collaborative usage rather than supply.
